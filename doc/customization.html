Customization Guide
Ganymede Release 1.0.10 - 2 April 2002

The Ganymede server has many advantages over the GASH project, including a graphical user interface, support for automated clients, and support for concurrent users. The biggest advantage, however, is that the Ganymede server is totally customizable through the authoring of custom classes and the visual editing of the Ganymede server's database schema.

Schema Editing

The Ganymede Admin console has as one of its primary functions the ability to edit the Ganymede Database Schema. You can access the schema editor when running the admin console as supergash. You will then see a window pop up that will look something like the following:

By clicking the right mouse button in the tree on the left of the schema editor window, you can create new object types, add, change or remove field definitions, and specify general parameters for field definitions. You can also associate custom code modules with particular object types in the database. The tree also allows you to drag fields up and down with the left mouse button to change the order in which fields are displayed in the client.

Custom plug-ins for object management

You can look at the schema source code included in release 0.96 to see examples of custom plug-in objects to manage objects in the server.

In order to have the server recognize plug-ins, you need to:

  • Modify the schema for the object type you wish to manage, giving a fully qualified class name for the class. i.e., arlut.csd.ganymede.custom.ldapEntryCustom
  • Compile your custom class and bind it into a jar file for the server. The build and buildCustomJar scripts in the schema directory will do this work for you.
  • Shut down the server from the Admin console so that the server gets a chance to do a full dump of the database file. The journal file does not maintain information about schema changes.
  • Restart the server with your updated custom.jar file in place

You may wish to consult the following javadoc's for details on the DBEditObject base class that is subclassed to provide object intelligence on the server.

  • The DBEditObject base class
  • The GanymediatorWizard class

We are also developing a guide to DBEditObject subclass construction here.

Custom plug-ins for scheduled and on-demand tasks

Finally, the server supports the authoring of custom tasks that can be run at regular intervals or on demand when a user commits a transaction in the database.

Again, you may wish to consult the javadoc's for documentation on the recommended base class for builder tasks. You can read about the server task registration and scheduling mechanism here.
