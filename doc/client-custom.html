Implementing a custom Ganymede client

Creating a custom client for the Ganymede server is very simple. All of the work involved in connecting to and communicating with the server is done through 2 classes: the ClientBase and the ClientListener.

Necessary Classes

The ClientBase class

The groundwork for connecting to the Ganymede server is implemented in the ClientBase. The ClientBase has two main methods for communicating with the server:

Session login(String username, String password) Start a new Session on the Ganymede server.
void disconnect() Close the Session.

login starts up a new Ganymede session, and disconnect finishes a session. All other communication is handled through the ClientListener interface.

The ClientListener interface

The ClientListener handles the communication from the server to the client. Only two methods are defined:

messageReceived(ClientEvent e) Called when the server needs to send a message to the client.
void disconnected(ClientEvent e) Called when the client is disconnected. This is for abnormal disconnects, not the disconnect() method in ClientBase.

Creating a client

In order to instantiate a new ClientBase, one must provide a string representing the Ganymede server URL and an object that implements the ClientListener interface. After creating the ClientBase object, login is used to get a referrence to the Session object. The Session object provides the interface to the server for future operations.

Example: PasswordClient

The PasswordClient is a simple text based program that emulates the Unix command passwd.
